маю цікаве поповнення в зомбікаліптичний колекції — project zomboid. про цей ігровий проект я вже кілька разів коротенько звітував у гуглоплюсах:
цитата: «…а ось на цей інді-проект я давно облизуюся… ну як, як можна було придумати і зробити настільки очевидно класнючу річ? адже перше правило ігробудування, якщо дивитися на те, що випускається — уникати всього очевидно класнючого. чому класне? тому що а) сурвайвал, б) зомбі, в) пісочниця» (читати повністю)
аж ось учора нарешті не витримав, придбав… 5,99 євро на desura — фактично заплатив 66,11 грн через paypal. наразі актуальна версія — 0.2 rc2.5, цілком іграбельне на windows та linux (перевірено на обох, хоча інсталятор для linux містить помилку і потребує трішки нетривіального «лікування», про це далі).

ті, кому потрібна завершена гра, можуть не перейматися: на цей момент забавка в активній розробці, зокрема версія 0.2 rc2.5 є, по суті, лише пісочницею — бракує сюжетного режиму і дуже багатьох цікавих можливостей, котрі повинні з’явитися в наступному реліз-кандидаті 3.0, або й пізніше…

…але навіть як пісочниця — project zomboid вартий уваги, і виявився дуже заразним. формат гри — рольова вешталка від третьої особи в ізометричній проекції зі спрайтовою графікою. головний персонаж опиняється в невеличкому американському містечку просто посеред локального зомбікаліпсису, і має вижити якомога довше, переховуючись в покинутих будівлях, збираючи та використовуючи різноманітні предмети, намагаючись за необхідності відбиватися від зомбі. сюжетна кампанія, можливо, матиме квести та якесь логічне завершення, але в режимі пісочниці можливий лише один варіант завершення гри… ну, ви зрозуміли.

де взяти і як придбати?

по-перше, можна спробувати демо-версію, але вона досить стара (0.1.4c) і в порівнянні з поточною — нецікава, а по-друге, інсталятор демки лише для windows.

по-друге, купуємо: desura підтримує оплату через paypal, тому в мене проблем не виникло. після оплати desura дозволяє зкопіювати свій ключ (вони це називають cd key), який виглядає як п’ять груп символів, розділених дефісами:

XXYYX-XYXYX-XXXYX-YXXXX-XXXXX

цей ключ знадобиться один раз, щоби запустити перший раз project zomboid, — тобто йоно не доведеться тримати під рукою. на всяк випадок його можна пригадати, відкривши історію покупок на сайті desura (лінк працює лише для зареєстрованих і залогінених користувачів).

далі — завантажуємо інсталятор реліз-кандидата 0.2.rc2.5 з сайту project zomboid, тому що на сайті desura наразі опублікована стабільна версія (0.1.5d), а отже — стара й малоцікава. ясно, що треба вибрати варіант для своєї системи (windows, linux чи macos).

як встановити і запустити?

детальні підказки щодо встановлення та налаштування під трьома операційними системами можна знайти в розділі питань та відповідей (faq) на форумі гри (звісно ж англійською).

deb-пакунок поточної версії для linux (принаймні 32-бітної, іншу не пробував) містить невеличку помилку, яка не дозволить його встановити, — але є «пігулка» (коротка інструкція англійською):
  1. в одну теку завантажити deb-пакунок project zomboid та пару bash-скриптів з ось цього архіву;
  2. залежно від версії deb-пакунка (32- чи 64-бітний) запустити відповідний скрипт (в моєму випадку rc2.5fix.32.bash);
  3. встановити забавку з пофіксеного пакунка projectzomboid_fixed.deb.
після цього скрипт для запуску гри projectzomboid.sh можна знайти в теці /opt/projectzomboid та самотужки додати в меню запуску. або ж не додавати =)

запустивши вперше гру на своєму старенькому і не надто потужному ноутбуці зcrunchbang linux, я отримав приблизно 1-5 кадрів на секунду і дуже засмутився. але парочка невеликих поправок в скрипті projectzomboid.sh (позначив кольором) дозволила грати практично без проблем:
# XMODIFIERS is cleared here to prevent SCIM screwing up keyboard input
XMODIFIERS= java \
    -Djava.library.path="${INSTDIR}" \
    -Dorg.lwjgl.util.NoChecks=true \
    -Dorg.lwjgl.librarypath="${INSTDIR}" \
    -Dsun.java2d.noddraw=true \
    -Xms1024m \
    -Xmx1024m \
    -XX:-UseSplitVerifier \
    -cp "${JARPATH}" \
    zombie.core.LaunchDialog \
    "$@" \
    >"${LOGFILE}"
на трішки потужнішому ноутбуці з windows проблем не виникло, але коли що — є і для windows відповідні підказки, тільки зауважу, що з деякими запропонованими у faq’ах параметрами в мене програма взагалі відмовлялася стартувати, потрібно експериментувати. окрім того натискання g в грі дозволяє переключити режим освітлення від красивішого, але повільнішого — до простого підсвічення клітинок, яке вимагає менше процесорного часу.
